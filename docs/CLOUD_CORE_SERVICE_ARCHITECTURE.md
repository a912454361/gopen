# G open 云端核心服务架构

## 一、架构概览

### 1.1 设计理念

**核心原则**：核心算法云端化，用户完全掌控，源码保护与开放性并存。

```
┌────────────────────────────────────────────────────────────────────┐
│                          架构设计理念                               │
├────────────────────────────────────────────────────────────────────┤
│  ✅ 用户完全掌控：部署、数据、密钥、资金全归用户                      │
│  ✅ 源码保护：核心AI服务云端化，SDK仅提供API接口                      │
│  ✅ 安全隔离：代码使用者无法反向控制系统                              │
│  ✅ 高可用性：分布式架构，多节点容错                                  │
│  ✅ 弹性扩展：按需扩容，支持高并发                                    │
└────────────────────────────────────────────────────────────────────┘
```

### 3.2 API 接口规范

**基础路径**：`https://cloud.gopen.ai/api/v1`

**认证方式**：
```typescript
// 请求头
Headers: {
  "Authorization": "Bearer <API_KEY>",
  "X-Device-ID": "<DEVICE_ID>",
  "X-Domain": "<YOUR_DOMAIN>",
  "Content-Type": "application/json"
}

// 签名算法
const signature = HMAC_SHA256(
  `${method}\n${path}\n${timestamp}\n${body}`,
  SECRET_KEY
);
```

**统一响应格式**：
```typescript
// 成功响应
{
  "code": 200,
  "message": "Success",
  "data": { ... },
  "timestamp": "2024-01-01T00:00:00Z",
  "requestId": "req_xxx"
}

// 错误响应
{
  "code": 400,
  "message": "Error message",
  "error": {
    "type": "ValidationError",
    "details": { ... }
  },
  "timestamp": "2024-01-01T00:00:00Z",
  "requestId": "req_xxx"
}
```

**错误码规范**：
```
1000-1999: 客户端错误
  1001: 参数错误
  1002: 认证失败
  1003: 授权无效
  1004: 资源不存在
  1005: 请求过于频繁

2000-2999: 服务端错误
  2001: 内部错误
  2002: 服务不可用
  2003: 超时
  2004: 第三方服务错误

3000-3999: 业务错误
  3001: 配额不足
  3002: 功能未授权
  3003: 任务不存在
  3004: 文件过大
```
